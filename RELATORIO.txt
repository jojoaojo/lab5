3. ARQUITETURA IMPLEMENTADA
═══════════════════════════════════════════════════════════════════════════════

✅ SOLUÇÃO: ARQUITETURA EM CAMADAS (LAYERED ARCHITECTURE)

┌─────────────────────────────────────────────────────────────────────┐
│                     PROTOCOLO DE COMUNICAÇÃO                        │
│                      (HTTP REST / gRPC)                             │
└─────────────────────────────────────────────────────────────────────┘
                                 ↓
┌─────────────────────────────────────────────────────────────────────┐
│                      CAMADA DE ADAPTAÇÃO                            │
│  ┌──────────────────────┐         ┌──────────────────────┐         │
│  │  RestUsersResource   │         │ GrpcUsersController  │         │
│  │  (extends            │         │ (extends             │         │
│  │   RestResource)      │         │  GrpcController)     │         │
│  └──────────────────────┘         └──────────────────────┘         │
│           ↓                                  ↓                      │
│  unwrapResultOrThrow()            toGrpcResult()                   │
└─────────────────────────────────────────────────────────────────────┘
                                 ↓
┌─────────────────────────────────────────────────────────────────────┐
│                 INTERFACE AGNÓSTICA (Users)                         │
│                   Result<T> postUser(User user)                     │
│                   Result<User> getUser(...)                         │
│                   Result<User> updateUser(...)                      │
│                   Result<User> deleteUser(...)                      │
│                   Result<List<User>> searchUsers(...)               │
└─────────────────────────────────────────────────────────────────────┘
                                 ↓
┌─────────────────────────────────────────────────────────────────────┐
│              LÓGICA DE NEGÓCIO (JavaUsers)                          │
│                                                                     │
│  ✓ Validação de dados                                              │
│  ✓ Persistência com Hibernate                                      │
│  ✓ Regras de negócio                                               │
│  ✓ Retorna Result<T> (OK ou ErrorCode)                             │
│  ✓ SEM dependência REST/gRPC                                       │
└─────────────────────────────────────────────────────────────────────┘
                                 ↓
┌─────────────────────────────────────────────────────────────────────┐
│                   PERSISTÊNCIA (Hibernate)                          │
│                      Base de Dados HSQLDB                           │
└─────────────────────────────────────────────────────────────────────┘


🔑 COMPONENTES-CHAVE:

1. Result<T> - Abstração de Resposta
   ─────────────────────────────────────
   Substitui exceções por objetos Result:
   
   interface Result<T> {
       boolean isOK();
       T value();
       ErrorCode error();
   }
   
   Uso:
   if (result.isOK())
       return result.value();
   else
       // tratar result.error()

2. JavaUsers - Lógica Centralizada
   ──────────────────────────────────
   UMA ÚNICA implementação para:
   ✓ Validar dados de entrada
   ✓ Persistir/buscar/atualizar/deletar
   ✓ Aplicar regras de negócio
   ✓ Retornar Result<T>
   
   SEM conhecimento de REST ou gRPC!

3. RestResource / GrpcController - Adapters
   ──────────────────────────────────────────
   Convertem Result<T> para o protocolo específico:
   
   REST:  Result<T> → HTTP Status Code + JSON
   gRPC:  Result<T> → gRPC Status + Protobuf Message

4. Factory Pattern
   ─────────────────
   Abstrai criação de clientes:
   
   Users client = UsersClientFactory.get("domain");
   // Retorna RestUsersClient OU GrpcUsersClient
   // baseado na URI descoberta

✅ SERVIDOR - ADAPTER REST
   ─────────────────────────
   
   📄 RestResource.java (BASE CLASS)
      ✓ unwrapResultOrThrow(Result<T>)
        - Converte Result OK → retorna value()
        - Converte Result ERROR → throw WebApplicationException
      
      ✓ errorCodeToStatus(ErrorCode)
        - NOT_FOUND → 404
        - CONFLICT → 409
        - FORBIDDEN → 403
        - BAD_REQUEST → 400
        - etc.
   
   📄 RestUsersResource.java (RESOURCE)
      Métodos SIMPLIFICADOS:
      
      public String postUser(User user) {
          return super.unwrapResultOrThrow(impl.postUser(user));
      }
      
      • Apenas chama JavaUsers
      • Delega conversão para RestResource
      • SEM lógica de negócio
      • 1 linha por método!

✅ SERVIDOR - ADAPTER gRPC
   ─────────────────────────
   
   📄 GrpcController.java (BASE CLASS)
      ✓ toGrpcResult(responseObserver, Result<T>, Function<T,R>)
        - Converte Result OK → onNext() + onCompleted()
        - Converte Result ERROR → onError()
      
      ✓ errorCodeToStatus(ErrorCode)
        - NOT_FOUND → NOT_FOUND
        - CONFLICT → ALREADY_EXISTS
        - FORBIDDEN → PERMISSION_DENIED
        - BAD_REQUEST → INVALID_ARGUMENT
        - etc.
   
   📄 GrpcUsersController.java (CONTROLLER)
      Métodos SIMPLIFICADOS:
      
      public void postUser(GrpcUser user, StreamObserver<...> resp) {
          super.toGrpcResult(resp, 
              impl.postUser(GrpcUser_to_User(user)),
              (addr) -> PostUserResult.newBuilder()
                          .setUserAddress(addr).build());
      }
      
      • Apenas chama JavaUsers
      • Converte tipos (GrpcUser ↔ User)
      • Delega conversão para GrpcController

✅ CLIENTES - BASE CLASSES
   ────────────────────────
   
   📄 RestClient.java (BASE)
      ✓ reTry(Supplier<Result<T>>)
        - Retry automático (3 tentativas)
        - Espera 10 segundos entre tentativas
        - Captura ProcessingException (timeout)
      
      ✓ processResponse(Response, Class<T>)
        - Converte HTTP Response → Result<T>
        - Mapeia status codes → ErrorCode
      
      ✓ Configuração:
        - Timeouts (READ: 10s, CONNECT: 10s)
        - Jersey Client pronto para uso
   
   📄 GrpcClient.java (BASE)
      ✓ processResponse(Supplier<T>)
        - Executa operação gRPC
        - Captura StatusRuntimeException
        - Converte gRPC Status → ErrorCode
      
      ✓ Configuração:
        - ManagedChannel com retry habilitado
        - Plaintext para desenvolvimento

✅ CLIENTES - IMPLEMENTAÇÕES
   ───────────────────────────
   
   📄 RestUsersClient.java (implements Users)
      Todos os 7 métodos implementados:
      
      public Result<String> postUser(User user) {
          return super.reTry(() -> doPostUser(user));
      }
      
      private Result<String> doPostUser(User user) {
          Response r = target.request()
              .accept(MediaType.APPLICATION_JSON)
              .post(Entity.entity(user, MediaType.APPLICATION_JSON));
          return super.processResponse(r, String.class);
      }
      
      • Usa reTry para todas as chamadas
      • Processa respostas com processResponse
      • Retorna Result<T> consistente
   
   📄 GrpcUsersClient.java (implements Users)
      Todos os 7 métodos implementados:
      
      public Result<String> postUser(User user) {
          return super.processResponse(() -> 
              stub.postUser(User_to_GrpcUser(user))
                  .getUserAddress()
          );
      }
      
      • Usa stub gRPC (blocking)
      • Converte tipos Protobuf ↔ User
      • Retorna Result<T> consistente

✅ FACTORY PATTERN
   ────────────────
   
   📄 UsersClientFactory.java (ESPECÍFICO)
      static Users get(String domain) {
          URI uri = Discovery.knownUrisOf(...)[0];
          return newClient(uri);
      }
      
      private Users newClient(URI uri) {
          if (uri.getPath().endsWith("/rest"))
              return new RestUsersClient(uri);
          if (uri.getPath().endsWith("/grpc"))
              return new GrpcUsersClient(uri);
          throw new RuntimeException("Unknown type");
      }
      
      • Detecta automaticamente REST vs gRPC
      • Retorna Users (interface comum)
   
   📄 GenericClientFactory.java (GENÉRICO)
      public class GenericClientFactory<T> {
          Function<URI, T> restClientFunc;
          Function<URI, T> grpcClientFunc;
          
          public T get(String domain) { ... }
      }
      
      Uso:
      var factory = new GenericClientFactory<>(
          Users.SERVICE_NAME,
          RestUsersClient::new,
          GrpcUsersClient::new
      );
      Users client = factory.get("domain");
      
      • Reutilizável para qualquer serviço
      • Usa Function<URI, T> para flexibilidade
